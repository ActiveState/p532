Virtual Environments

ActivePerl is Installed via the command line using our State Tool CLI, which will automatically create a virtual environment to ensure against dependency conflicts and environment corruption.

No Compiler Toolchain

Need to add specific dependencies post-install? We’ll automatically build them (including linked C libraries) for you, just like PPM. No need to maintain your own build environment.

Support for Multi-OS Teams

Share your pre-built Perl environment with your team members, no matter whether they use Windows, Mac or Linux, ensuring everyone’s environment is in sync.

Track & Restore Environments

The ActiveState Platform saves each change you make to your environment so you can roll back/restore any version at any time.

No More Dependency Hell

Automatically resolves all dependencies, and provides clear messaging when conflicts arise, simplifying resolution..

Vetted Perl

Our catalog of Perl modules is imported from CPAN, vetted by ActiveState, and updated every 48 hours to ensure you’re always building with the latest dependency versions. Know the provenance of what you’re installing.
